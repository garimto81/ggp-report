## 프로젝트 개요

### 프로젝트명
**Camera Work Auto Report** - Firebase/GitHub 기반 카메라 파트 자동 업무 보고 시스템

### 목적
카메라 파트(담당자: Aiden Kim)의 일일 업무 보고서를 Firebase와 GitHub 데이터를 기반으로 완전 자동화하여, AI 기반 우선순위 분석을 통해 Google Docs에 자동으로 작성합니다.

### 대상
- **주 사용자**: 카메라 파트 Aiden Kim
- **보고서 열람자**: 경영진 및 프로젝트 관리자

### 개발 기간
- 2024년 12월 ~ 2025년 1월 (v1.0 출시)
- 2025년 2월 ~ 현재 (지속적인 개선)

## 시스템 아키텍처

### 전체 구조
```
┌─────────────┐     ┌────────────────┐     ┌─────────────┐
│  Firebase   │────▶│ Report         │────▶│  Gemini AI  │
│  Firestore  │     │ Generator      │     │  Analyzer   │
└─────────────┘     └────────────────┘     └─────────────┘
       │                     │                      │
┌─────────────┐              │                      ▼
│   GitHub    │──────────────┘              ┌─────────────┐
│   API       │                             │ Google Docs │
└─────────────┘                             │   Writer    │
                                           └─────────────┘
```

### 주요 컴포넌트

#### 1. 데이터 수집 레이어
- **FirebaseDataFetcher**: Firestore에서 카메라 파트 업무 데이터 수집
- **GitHubDataFetcher**: GitHub 이슈, 커밋, PR 추적
- **싱글톤 패턴**: 효율적인 인스턴스 관리
- **캐싱 시스템**: 5분간 데이터 캐시

#### 2. AI 분석 엔진
- **GeminiAnalyzer**: Google Gemini AI 통합
- **우선순위 분석**: 비즈니스 임팩트, 긴급도, 의존성
- **상위 3개 선정**: 가장 중요한 업무 자동 선별
- **점수 시스템**: 1-100점 우선순위 점수

#### 3. 보고서 생성 시스템
- **GoogleDocsWriter**: Google Docs API 통합
- **날짜별 관리**: 특정 날짜 탭 자동 찾기
- **구조화된 형식**: 업무명, 진행률, 우선순위 이유
- **배치 업데이트**: 효율적인 문서 수정

#### 4. 자동화 및 스케줄링
- **GitHub Actions**: 완전 자동화된 실행
- **평일 실행**: 월-금 오전 10시 (KST)
- **재시도 메커니즘**: 실패 시 자동 재실행
- **수동 실행**: 필요시 즉시 실행 가능
